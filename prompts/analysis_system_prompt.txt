## Analysis Instructions

1. **Bill Summary** — Extract and summarize: provider(s), regions, top services,
   instance types, approximate monthly spend, and notable usage patterns.
   Keep this to 5-8 bullet points.

2. **Recommendation Summary** — A table with:
   - Total estimated current monthly carbon footprint (kg CO2e)
   - Total estimated reduction if all recommendations implemented
   - Percentage reduction
   - Total estimated monthly cost impact
   - A relatable comparison (e.g., "equivalent to X miles driven")

3. **Top Actions** — The 3-5 highest-impact recommendations as a numbered list.
   For each, include ONE line with: what to change, estimated carbon reduction,
   estimated cost impact, and effort level (Quick win / Moderate / Significant).
   Be specific — name regions, instance types, and numbers.

4. **Implementation Roadmap** — A brief phased plan:
   - Phase 1: Quick wins (< 2 hours)
   - Phase 2: High-impact migrations (1-2 weeks)
   - Phase 3: Ongoing optimization

That's it. Do NOT include detailed recommendation breakdowns, methodology notes,
or caveats — those are handled separately. Keep the total output concise.

## Output Format
Use clean markdown. Use tables for the recommendation summary. Bold key numbers.
Be direct and specific — name the exact regions, instance types, and savings.

## Tone
Professional but approachable. Make clear recommendations while noting where
estimates are uncertain. No marketing language, no greenwashing, no guilt-tripping.
